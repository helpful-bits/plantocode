## PSEO Page Template Structure

Each PSEO page includes:

1. **Metadata**
   - Meta title, description
   - Open Graph tags (og:title, og:description, og:url, og:type)
   - Canonical URL
   - Alternates/hreflang

2. **Content Sections**
   - Breadcrumbs navigation
   - Hero (headline + subhead)
   - Category badge
   - Pain points (problem/solution pairs)
   - Comparison table (if applicable)
   - Workflow steps (4-6 cards)
   - Key features (capabilities list)

3. **Technical Implementation**
   - File discovery explanation
   - Multi-model planning overview
   - Quick setup (3-step guide)

4. **Social Proof**
   - Success metrics (75%, 3x, 100%)
   - Customer testimonials section

5. **Internal Linking**
   - **Related Topics** (6+ contextual links based on category/tool/OS/language)
   - **Related Resources** (docs, demo, architecture)
   - **Download CTA** (with platform selection)

6. **Structured Data**
   - SoftwareApplication schema
   - HowTo schema (workflow steps as HowToSteps)
   - Application category
   - Operating systems
   - Offers/pricing

## Publishing & Maintenance

### Adding New Pages
1. Add entry to appropriate JSON file in `website/src/data/pseo/`
2. Set `publish: true` to include in sitemap
3. Build regenerates static params automatically
4. Page appears in sitemap.xml next deployment

### Updating Content
1. Edit JSON content (pain_points, workflow_steps, etc.)
2. Page template handles re-rendering automatically
3. No code changes needed (data-driven)

### Draft Management
1. Set `publish: false` to remove from sitemap
2. Page still renders (direct URL access possible)
3. Useful for pre-release or A/B testing
